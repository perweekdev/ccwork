# 태그 기능 정의서 (확정)

## 기능 개요

노트에 태그를 추가하고 관리할 수 있다.

## 데이터 구조

### Note 타입 변경

```ts
interface Note {
  id: string;
  title: string;
  content: string;
  createdAt: string;
  updatedAt: string;
  tags: string[]; // 추가
}
```

- 태그는 Note 객체 내 `string[]`로 저장한다.
- json-server `db.json`의 notes 항목에 `tags` 배열 필드 추가.
- 기존 노트의 `tags`는 `[]`로 초기화한다.

## 기능 요구사항

### 태그 추가

- NoteEditor 하단에 태그 입력 영역을 배치한다.
- 텍스트 입력 후 `Enter` 또는 `,`(쉼표) 입력 시 태그가 확정된다.
- 확정된 태그는 입력창 앞에 뱃지(chip) 형태로 표시된다.

### 태그 삭제

- 각 태그 뱃지에 `×` 버튼을 두어 클릭 시 해당 태그를 제거한다.

### 태그 저장

- 태그 변경(추가/삭제)은 기존 "저장" 버튼 클릭 시 title/content와 함께 일괄 저장된다.
- 저장 전 변경은 로컬 상태에만 반영되며, 저장하지 않고 다른 노트로 이동하면 변경이 사라진다.

### 태그 표시

- NoteEditor(상세 화면): 저장된 태그를 뱃지로 표시한다.
- NoteItem(사이드바): 해당 노트의 태그를 소형 뱃지로 표시한다.

## 유효성 규칙

| 규칙      | 내용                                                |
| --------- | --------------------------------------------------- |
| 최대 길이 | 20자 초과 시 입력 무시                              |
| 허용 문자 | 제한 없음 (한글, 영문, 숫자, 특수문자 모두 허용)    |
| 대소문자  | 소문자로 normalize (`React` → `react`)              |
| 공백 처리 | 앞뒤 trim + 내부 공백 제거 (`react js` → `reactjs`) |
| 중복      | 동일 태그 중복 추가 불가 (normalize 후 비교)        |
| 빈 입력   | 공백만 입력 후 확정 시 무시                         |
| 최대 개수 | 제한 없음                                           |

## 엣지 케이스

- 새 노트 생성 시 태그는 `[]`로 시작한다.
- 태그 입력 중 다른 노트를 선택하면 미확정 입력값은 버린다.
- normalize 결과가 기존 태그와 중복되면 조용히 무시한다 (에러 없음).

## 임의 결정 항목 (질문 생략)

- **태그별 노트 필터링**: 미구현. 이번 스펙 범위 밖.
- **태그 순서**: 추가 순서 유지.
- **자동완성**: 미구현. 이번 스펙 범위 밖.
