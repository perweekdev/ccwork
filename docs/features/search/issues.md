# 검색 기능 이슈 분해

## 수직 슬라이싱 원칙

각 이슈는 UI → 로직 → 데이터 레이어를 수직으로 관통해 사용자에게 측정 가능한 가치를 전달한다.
의존 순서: Issue 1 → Issue 2, Issue 3, Issue 4 (2·3·4는 병렬 가능)

## Issue 1: 검색창 UI & 실시간 필터링

**설명**  
사이드바에 검색 입력창을 추가하고, 입력할 때마다 `notes[]`를 제목·내용 기준으로 실시간 필터링한다. `searchQuery` 상태를 `NoteList` 로컬에서 관리하고, 검색창이 빈 문자열이면 전체 목록을 표시한다. 이후 이슈들의 선행 조건이다.

**완료 조건 (Acceptance Criteria)**

- `NoteList`에 `searchQuery: string` 로컬 state가 존재한다.
- 검색 입력창이 "New Note" 버튼 아래 사이드바 상단에 배치된다.
- 입력창의 placeholder가 `"노트 검색..."`이다.
- 입력할 때마다 `title`과 `content` 모두를 대상으로 필터링된다.
- 검색어와 노트 데이터를 모두 소문자로 변환 후 비교한다 (대소문자 무시).
- 검색창이 빈 문자열이면 전체 노트 목록이 표시된다.

**시나리오 1 — 키워드 입력 시 실시간 필터링**

_Given_ 노트 목록에 "React 입문", "Vue 기초" 두 노트가 있다  
_When_ 사용자가 검색창에 "react"를 입력한다  
_Then_ "React 입문" 노트만 목록에 표시된다

**시나리오 2 — 대소문자 무시**

_Given_ 노트 목록에 "React 입문" 노트가 있다  
_When_ 사용자가 검색창에 "REACT"를 입력한다  
_Then_ "React 입문" 노트가 목록에 표시된다

**시나리오 3 — 내용 기반 검색**

_Given_ 제목은 "오늘 학습"이고 내용에 "typescript"가 포함된 노트가 있다  
_When_ 사용자가 검색창에 "typescript"를 입력한다  
_Then_ "오늘 학습" 노트가 목록에 표시된다

**시나리오 4 — 검색창 비움 시 전체 목록 복귀**

_Given_ 검색창에 "react"가 입력되어 목록이 필터링된 상태다  
_When_ 사용자가 검색창을 모두 지운다  
_Then_ 전체 노트 목록이 다시 표시된다

## Issue 2: 검색 결과 없음 안내

**설명**  
필터링 결과가 빈 배열일 때 목록 자리에 안내 메시지를 표시해, 사용자가 결과 부재를 명확히 인지할 수 있게 한다.

**완료 조건 (Acceptance Criteria)**

- 필터링된 노트 목록이 빈 배열이면 "검색 결과가 없습니다." 텍스트가 표시된다.
- 안내 텍스트는 NoteItem 목록이 표시되던 자리에 렌더링된다.
- 검색창이 비어 있을 때(전체 목록 상태)에는 안내 텍스트가 표시되지 않는다.

**시나리오 1 — 일치하는 노트 없음**

_Given_ 노트 목록에 "React 입문", "Vue 기초" 두 노트가 있다  
_When_ 사용자가 검색창에 "angular"를 입력한다  
_Then_ NoteItem 목록 대신 "검색 결과가 없습니다." 텍스트가 표시된다

**시나리오 2 — 검색 결과 있을 때 안내 텍스트 미표시**

_Given_ 검색창에 "react"를 입력해 1개의 노트가 필터링된 상태다  
_When_ 사이드바가 렌더링된다  
_Then_ "검색 결과가 없습니다." 텍스트가 표시되지 않는다

## Issue 3: 검색 결과에서 노트 선택 & 검색어 유지

**설명**  
필터링된 목록에서 노트를 클릭해 열 수 있고, 노트를 선택한 후에도 검색어가 초기화되지 않고 유지된다.

**완료 조건 (Acceptance Criteria)**

- 필터링된 NoteItem 클릭 시 해당 노트가 NoteEditor에 열린다.
- 노트 선택 후 검색창의 검색어가 그대로 유지된다.
- 다른 노트를 선택해도 검색 필터 상태가 유지되어 목록이 동일하게 필터링된 채로 남는다.

**시나리오 1 — 검색 결과에서 노트 열기**

_Given_ 검색창에 "react"가 입력되어 "React 입문" 노트 하나가 표시된다  
_When_ 사용자가 "React 입문" NoteItem을 클릭한다  
_Then_ "React 입문" 노트가 NoteEditor에 열리고, 검색창에는 "react"가 그대로 유지된다

**시나리오 2 — 노트 선택 후 검색 필터 유지**

_Given_ "react"로 검색해 2개의 노트가 필터링된 상태에서 첫 번째 노트를 열었다  
_When_ 사용자가 두 번째 필터링된 노트를 클릭한다  
_Then_ 두 번째 노트가 열리고, 목록은 여전히 "react" 기준으로 2개만 표시된다

## Issue 4: 새 노트 생성 시 검색어 초기화

**설명**  
새 노트 버튼 클릭 시 검색어를 빈 문자열로 초기화해, 새로 생성되는 노트가 전체 목록 컨텍스트에서 열리도록 한다.

**완료 조건 (Acceptance Criteria)**

- "New Note" 버튼 클릭 시 `searchQuery`가 빈 문자열로 초기화된다.
- 검색어 초기화와 함께 전체 노트 목록이 즉시 복원된다.
- 초기화는 `NoteList` 내부에서 `onNewNote` 핸들러를 래핑해 처리한다. `App.tsx` 변경 없음.

**시나리오 1 — 검색 중 새 노트 생성**

_Given_ 검색창에 "react"가 입력되어 목록이 필터링된 상태다  
_When_ 사용자가 "New Note" 버튼을 클릭한다  
_Then_ 검색창이 비워지고 전체 노트 목록이 표시되며, 새 노트 에디터가 열린다

**시나리오 2 — 검색어 없을 때 새 노트 생성**

_Given_ 검색창이 비어 있는 상태다  
_When_ 사용자가 "New Note" 버튼을 클릭한다  
_Then_ 전체 목록이 유지된 채로 새 노트 에디터가 열린다 (동작 변화 없음)
