# 검색 기능 정의서 (확정)

## 기능 개요

사이드바 상단 검색창에 키워드를 입력하면 노트 목록이 실시간으로 필터링된다.

## 데이터 구조

- 별도 데이터 구조 변경 없음.
- 클라이언트 상태(`notes[]`)를 필터링하며 서버 요청 없음.
- 검색어는 컴포넌트 로컬 state로 관리 (`searchQuery: string`).

## 기능 요구사항

### 검색창

- NoteList 사이드바 상단, 기존 "New Note" 버튼 아래에 검색 입력창을 배치한다.
- placeholder: `"노트 검색..."`

### 필터링 동작

- 검색창에 입력할 때마다 실시간으로 NoteList를 필터링한다.
- 검색 범위: 노트 **제목 + 내용** 모두 포함 여부로 판단한다.
- 대소문자 구분 없음 (입력값과 노트 데이터 모두 소문자로 변환 후 비교).
- 검색창을 비우면 전체 노트 목록으로 돌아온다.

### 노트 선택

- 검색 결과에서 노트를 클릭하면 해당 노트를 열고, 검색어는 유지한다.
- 검색어를 직접 지우기 전까지 필터 상태가 유지된다.

### 검색 결과 없음

- 일치하는 노트가 없을 때 목록 자리에 `"검색 결과가 없습니다."` 안내 텍스트를 표시한다.

## 유효성 규칙

| 규칙           | 내용                            |
| -------------- | ------------------------------- |
| 대소문자       | 구분 없음 (소문자 변환 후 비교) |
| 앞뒤 공백      | trim 후 비교                    |
| 최소 입력 길이 | 제한 없음 (1자부터 필터링)      |

## 엣지 케이스

- 검색 중 새 노트 생성 버튼 클릭 시 검색어는 초기화한다.
- 검색 중 선택된 노트가 삭제되면 검색어는 유지하고 선택만 해제한다.

## 임의 결정 항목 (질문 생략)

- **검색어 하이라이트**: 미구현. 이번 스펙 범위 밖.
- **디바운스**: 클라이언트 필터링이므로 불필요.
- **태그 검색 연동**: 미구현. 이번 스펙 범위 밖.
