# Notes 디자인 시스템

> 현재 앱 화면을 기준으로 추출한 디자인 토큰과 규칙. 모든 스타일 작업 시 이 문서 체계를 참고한다.

## 디자인 원칙

| 원칙              | 설명                                                           |
| ----------------- | -------------------------------------------------------------- |
| **Simple**        | 복잡한 기능보다 메모 작성에 집중할 수 있도록 단순한 구조 유지  |
| **Focus**         | 선택된 노트에 집중할 수 있도록 좌측 목록 / 우측 상세 영역 분리 |
| **Soft UI**       | 부드러운 회색 계열 배경과 둥근 카드 사용                       |
| **Content First** | 장식보다 텍스트 가독성 우선                                    |

## 파일 구조

| 파일                             | 내용                                       | 참조 시점               |
| -------------------------------- | ------------------------------------------ | ----------------------- |
| [`tokens.md`](tokens.md)         | CSS 변수 ↔ Tailwind 매핑, 색상 스케일      | `src/index.css` 수정 시 |
| [`typography.md`](typography.md) | 폰트 패밀리, 텍스트 스케일                 | 텍스트 관련 작업 시     |
| [`components.md`](components.md) | Radius·Shadow 토큰, 컴포넌트별 클래스 패턴 | 컴포넌트 생성·수정 시   |
| [`rules.md`](rules.md)           | Do / Don't 규칙 (CLAUDE.md에 자동 로드)    | 항상 적용               |
| [`workflow.md`](workflow.md)     | Claude 사용 시나리오별 요청 흐름           | 작업 방법 모를 때       |

## 디자인 스킬

| 스킬                     | 용도                                     |
| ------------------------ | ---------------------------------------- |
| `/design-check <파일>`   | 규칙 위반 여부 점검                      |
| `/new-component <이름>`  | 디자인 시스템 패턴으로 컴포넌트 스캐폴딩 |
| `/token-add <이름> <값>` | 코드·문서 동시 토큰 추가                 |
